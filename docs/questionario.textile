

*1)* Responda as seguintes questões:

*1.2)* Descreva o que é um padrão.
 É uma maneira testada ou documentada de alcançar um objetivo qualquer. Os padrões são comuns em várias áreas da engenharia e não são invenções originais

*1.3)* Qual o principal objetivo dos padrões do Catálogo GRASP?
 Programar melhor com orientação a objetos, desenvolver softwares de qualidade, utilizando eficientemente polimorfismo , herança, modularidade, composição, abstração para construir código reutilizável, eficiente de alta coesão e baixo acoplamento.

*1.4)* Descreva brevemente o padrão Criador.
 Muitas das vezes é aplicado ao objeto que conhece os dados iniciais do objeto a ser criado. São objeto com responsabilidade de criar outros objetos.

*1.5)* Descreva brevemente o padrão Especialista.
 É um objeto que deve ter conhecimentos para realizar determinadas tarefas, exemplo, calcular o valor total de uma venda.  Reúne informações de outras classes ‘leves’  e mantém o comportamento distribuído, favorecendo a alta coesão.

*1.6)* Descreva brevemente o padrão Alta Coesão.
 Classes que mantém tarefas que fazem parte da proposta a qual foram criadas, que não faz coisas não relacionadas à outras classes.
